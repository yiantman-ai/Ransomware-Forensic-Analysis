# Contributing to Ransomware Forensic Analysis

Thank you for your interest in contributing to this educational project!

## Ways to Contribute

### 1. Improve Documentation
- Fix typos or unclear explanations
- Add more detailed analysis
- Translate documentation
- Add diagrams or visualizations

### 2. Enhance Analysis
- Add new detection rules
- Improve timeline accuracy
- Expand MITRE ATT&CK mapping
- Add additional forensic techniques

### 3. Add Tools
- Create new analysis scripts
- Improve existing tools
- Add automation capabilities
- Develop visualization improvements

### 4. Report Issues
- Documentation errors
- Broken links
- Analysis inconsistencies
- Tool bugs

## Contribution Guidelines

### Before Contributing

1. **Check existing issues** - Someone might already be working on it
2. **Open an issue** - Discuss your proposed changes
3. **Follow the code of conduct** - Be respectful and professional

### Making Changes

1. **Fork the repository**
```bash
   git clone https://github.com/yiantman-ai/Ransomware-Forensic-Analysis.git
   cd Ransomware-Forensic-Analysis
```

2. **Create a branch**
```bash
   git checkout -b feature/your-feature-name
```

3. **Make your changes**
   - Follow existing code style
   - Add comments where needed
   - Update documentation

4. **Test your changes**
   - Verify tools work correctly
   - Check for broken links
   - Validate JSON/CSV formats

5. **Commit with clear messages**
```bash
   git commit -m "Add: New Snort detection rule for C2 traffic"
```

6. **Push and create Pull Request**
```bash
   git push origin feature/your-feature-name
```

### Code Style

**Python:**
- PEP 8 compliance
- Docstrings for functions
- Type hints where appropriate
- Comments for complex logic

**Markdown:**
- Clear headings hierarchy
- Code blocks with language tags
- Tables for structured data
- Consistent formatting

**Detection Rules:**
- Clear rule descriptions
- Severity classifications
- False positive considerations
- Testing notes
